💡 Our plans for more features

We at Orlovsky Consulting GbR very excited and happy about current progress of custom T2 Linux build.

At the moment we still figuring out , what is the best approach for working on Linux desktop.

I Alexander Orlovsky CEO of Orlovsky Consulting GbR , in my personaly opinion the modern Linux desktop is big there multiple different approaches and it is hard to satisfy all needs and tastes, but what seems more reasonable and realistic to integrate with.

  • Trinity desktop

  • XFCE

Only those 2 window managers in my personal opinion are most reliable UI , sure some people would prefer modern KDE or GNOME , but i find them and i mean theirs community of developers hard to work and hard to collaborate and hard to communicate.

What we at custom T2 Linux wouldn't do, that to include very ambgios window managers, which require big build pipelines and even bigger building rigs.

WWhat about steam , wine support ? I think gaming should be possible , but you need to understand first we build window managers , test them , make sure they are all stable

Vanila upstream T2 Linux is very rollable, meaning they always on bleeding edge, but custom T2 Linux build by definition we and our intetion to be a stable base , we just not blindly add packages with hope that it works. We care about quality and added packages only then when we assured and safe that it wont break any other package dependencies.

So here is our plan

  • 1. Community decides which X server we should integrate.
  • 2. We start building custom Linux desktop build as we at Orlovsky Consulting GbR planned.
  • 3. We integrate with TDE window manager.
  • 4. We integrate with XFCE window manager.
  • 5. We start building everything which needed for steam and wine software support.
  • 6. We start to promote our version of modern Linux desktop worldwide.
  • 7. We ignore KDE and or Gnome dev community and any Linux corporation, because they tend to be very tribal this days.
  • 7.1 We find and cooperate with any other developer/s or Linux distribution/s which share our technology views and values.
  • 8. Finally we collect feedback from community and our commercial clients.
  • 9. We promote opensource and cooperation between different opensource projects, we ensure that our solution for Linux desktop stays stable and adaptable for any future user feature requests.

The end game should be we finally put very nice and quality desktop Linux , which runs on a lot of different x86_64 platforms and use huge amount of freely available packages
